API Usage

Send your API key in the X-API-Key header:

curl -H "X-API-Key: esa_xxxxxxxx.your_secret_here" \
     https://backend.extraspace.com.sg/api/discount-plans
Response headers: Each API response includes rate limit and quota information:
  • X-RateLimit-Limit — your per-minute rate limit
  • X-Quota-Limit — your daily quota
  • X-Quota-Remaining — remaining requests for today

API Reference — showing endpoints for your assigned scopes

How to choose the right endpoint group
  • 🤖 Chatbot Booking Flow — purpose-built for the chatbot & recommendation engine. Pricing is computed by the engine, idempotent move-ins, automatic SOAP orchestration for perpetual+prepay plans, and session/customer reconciliation back to the recommendation that produced the booking.
  • 📦 SiteLink Reservations (Direct) — thin passthrough to SiteLink for CRUD on existing reservations (list, update, cancel, notes, fee config, etc.) Use when you're not going through the recommender — e.g. ops tooling, back-office integrations, manual reservations.
  • Inventory / Discount Plans / Smart Lock / Statistics / Reservation Fees — supporting data sources. Read-mostly references that any caller can use.
Chatbot booking — 4 calls in order:
  1. POST /api/recommendations — quote engine; read pricing.total_due_at_movein + customer_disclosure.fine_print.
  2. POST /api/reservations/reserve — hold the unit; pass session_id + customer_id for outcome reconciliation.
  3. GET /api/reservations/move-in/cost — SOAP-truth confirmation right before charging.
  4. POST /api/reservations/move-in — pass the total as payment_amount. Send Idempotency-Key: <UUID> to make retries safe (24h cache; replays return idempotent_replay: true).
Perpetual+prepay plans: the middleware fans out the follow-up SOAP calls internally (PaymentSimpleCash for the prepay surplus, ScheduleTenantRateChange for the future ECRI). Bot's contract stays single-amount. See chatbot_integration.md for the full lifecycle.
